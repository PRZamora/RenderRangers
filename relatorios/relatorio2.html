1. Introdução

Os objetivos do trabalho eram:

  • Elaborar um programa que faça os filtros média, gaussiano, mediana, e bilateral em uma imagem.
  • Usar esse programa com diferentes Kernel (3x3, 5x5, 7x7, e 11x11)
  • Elaborar um novo programa que execute os filtros acima, porém a entrada da imagem será a webcam e a imagem nova com filtro será salva ao clique da tecla [s]. O kernel e filtro ficam a escolha do programador.

2. Fundamentos Básicos

Para a execução do laboratório foi necessário a utilização do programa OpenCV.

Para desenvolvimento dos programas foi necessário entender a teoria por trás dos filtros aplicados, assim como conhecimentos em C++ para aplicar tais teorias no código.

3. Materiais e Métodos

Os principais materiais e métodos utilizados neste laboratório foram:

  • Biblioteca OpenCV para captura e manipulação de imagens e vídeos.
  • Ambiente de desenvolvimento configurado com o compilador GCC e CMake para a execução dos exemplos em C++.
  • Procedimentos experimentais que incluíram a leitura e exibição de imagens e vídeos, de modo a aplicar filtros nas imagens conforme passo a passo descrito na página do OpenCV pelo link: OpenCV Filtros. As imagens utilizadas foram as mesmas utilizadas no Relatório 1.
  • Aplicação dos programas nas fotos escolhidas e salvamento dos resultados.

Também tivemos que executar uma versão desse código que fosse aplicada diretamente à webcam. Para cumprir este item 3, construímos a seguinte versão do código:

Nome do Script: FilterWebcamImage.cpp


                #include 
                #include 
                #include 
                #include 
                
                using namespace std;
                using namespace cv;
                
                int KERNEL_SIZE = 11; // Não será mais input
                Mat frame, dst;
                char current_filter = 'a';  // Filtro inicial: média (blur)
                
                void apply_filter(char filter) {
                    switch (filter) {
                        // Era pra escolher dois filtros. Porém, mais fácil "iniciar" o desafio e alternar entre todos eles de uma vez
                        // dado que os comandos são os mesmos
                        case 'a':  // Média
                            blur(frame, dst, Size(KERNEL_SIZE, KERNEL_SIZE), Point(-1, -1));
                            break;
                        case 'g':  // Gaussiano
                            GaussianBlur(frame, dst, Size(KERNEL_SIZE, KERNEL_SIZE), 0, 0);
                            break;
                        case 'm':  // Blur Mediano
                            medianBlur(frame, dst, KERNEL_SIZE);
                            break;
                        case 'b':  // Bilateral
                            bilateralFilter(frame, dst, KERNEL_SIZE, KERNEL_SIZE * 2, KERNEL_SIZE / 2);
                            break;
                        default:
                            dst = frame.clone();  // Sem filtro
                            break;
                    }
                }
                
                int main() {
                    VideoCapture cap(0);  // Captura da webcam
                    if (!cap.isOpened()) {
                        cerr << "Erro ao abrir a câmera!" << endl;
                        return -1;
                    }
                
                    namedWindow("Filtro em Tempo Real", WINDOW_AUTOSIZE);
                
                    while (true) {
                        cap >> frame;  // Captura o frame atual da webcam
                        if (frame.empty()) {
                            cerr << "Erro ao capturar frame da câmera!" << endl;
                            break;
                        }
                
                        apply_filter(current_filter);  // Aplica o filtro escolhido
                
                        imshow("Filtro em Tempo Real", dst);  // Mostra o frame filtrado
                
                        char key = (char)waitKey(30);  // Espera por uma tecla por 30ms
                        if (key == 's') {  // Tecla 's' para salvar a imagem
                            string filename = "imagem_wc_filtrada.jpg";
                            imwrite(filename, dst);
                            cout << "Imagem salva como " << filename << endl;
                        } else if (key == 'a' || key == 'g' || key == 'm' || key == 'b' || key == 'o') {
                            current_filter = key;  // Troca de filtro
                        } else if (key == 27) {  // Tecla 'ESC' para sair. Estava me irritando ao tentar fechar a janela e ela reabrir
                            break;
                        }
                    }
                
                    cap.release(); // Finalizando captura de wc
                    destroyAllWindows(); // Garantir que se encerre a janela
                    return 0;
                }

5. Conclusões e Comentários Finais

Neste laboratório, pudemos explorar a aplicação através do OpenCV de diferentes filtros de suavização em imagens, a saber: filtro de média, gaussiano, mediana e bilateral. Pudemos também observar os resultados do filtro quando aplicados em uma imagem com ruído, e analisar os efeitos que o filtro causa sobre o ruído aplicado na imagem original.

Na aplicação dos filtros, observamos uma relação direta entre o tamanho do kernel, e a intensidade do efeito de filtro. Para as imagens sem o ruído aplicado, observamos que os filtros com kernel 3x3 não resultam em diferenças notórias na imagem; os efeitos passam a ser perceptíveis somente a partir do kernel 5x5. O filtro de mediana apresentou resultado mais intenso, e o filtro bilateral apresentou resultado mais sutil. Por outro lado, nas imagens com o ruído aplicado, os filtros de média, mediana e gaussiano de kernel 3x3 já foram suficientes para obtermos resultados notáveis, suavizando os efeitos do ruído e sem perder muito da informação original da imagem. A partir do tamanho de kernel 5x5, os resultados foram bastante semelhantes aos obtidos com a imagem sem ruído. Já para o filtro bilateral, seu efeito foi muito pouco perceptível para qualquer tamanho de kernel, tanto na imagem original quanto na imagem com ruído.

Os resultados obtidos reforçam a importância de uma escolha adequada de filtro e tamanho de kernel para as diferentes necessidades de processamento de vídeo e imagens, ao nos mostrar como um mesmo filtro pode ter resultados diferentes dependendo das condições iniciais da imagem (com ou sem ruído).
